# Spec: Typing Everywhere 增强交互与设置

## 已确认假设

1. 本次是在现有 `Typing Everywhere` 油猴脚本上做增量增强，不重写整体架构。
2. 继续使用脚本自有的 Shadow DOM UI 和原位覆盖层，不把目标段落拆成业务 DOM 的彩色 span。
3. 目标段落在练习期间允许被临时隐藏并由覆盖层模拟显示，但退出后必须恢复原始外观与可交互性。
4. 新的颜色配置只作用于脚本覆盖层、选段框线和辅助 UI，不修改原网页样式表。
5. `Tab` / `Shift+Tab` 的跳过行为优先于浏览器原生焦点切换，只在打字模式内拦截。
6. 跳过字符和跳过整段都不计入速度统计，也不计入错误率分母。
7. 自动隐藏入口图标只在空闲状态触发；只要用户正在练习、正在选段或正在操作设置，就不得自动缩边。

## Objective

在当前脚本基础上补足更接近打字练习器的完整交互：进入选段前给出醒目的中央提示；入口图标扩展出设置面板；支持可配置的字符颜色、框线颜色和滚动跟随开关；支持 `Tab` 跳过当前字符、`Shift+Tab` 跳过当前段落；并优化图标的拖动、边距、自动隐藏与图标样式。

目标不是把脚本做成“平台”，而是在现有用户流上增加必要的可见反馈和少量高价值设置，使用户在复杂网页上也能自然练习。

## 用户流程

1. 页面加载后显示新的打字练习入口图标，默认停在右上角安全边距内。
2. 用户左键点击主图标后进入 `selecting` 模式，屏幕中央出现“请选择一段文本，Esc 退出”的醒目提示。
3. 用户悬停网页文本时，最近语义文本段显示可配置颜色的框线。
4. 用户点击段落后进入 `typing` 模式：
   - 原段落临时隐藏
   - 覆盖层在原位置渲染文本
   - 未打字符显示未打色
   - 打对字符显示正确色
   - 打错字符显示错误色
   - 跳过字符显示跳过色
5. 用户按 `Tab` 跳过当前单字符；按 `Shift+Tab` 跳过当前段剩余字符，并短暂显示跳过预览后切到下一段。
6. 用户可通过设置按钮展开面板，选择预设主题、修改高级颜色，并切换“是否始终跟随当前打字段到屏幕中心”。
7. 用户按 `Esc` 退出选段或打字模式，中央提示、覆盖层和统计面板清理，原网页恢复。
8. 若入口 5 分钟未使用且当前为 `idle`，图标自动缩到边缘，只留可见把手；鼠标靠近时滑出。

## Functional Requirements

### 入口图标与空闲行为

- 主图标与设置按钮组成一个入口组件。
- 主图标只响应左键开始选段；右键不得触发拖动、开始练习或浏览器原生右键菜单。
- 拖动只对主图标左键生效；设置按钮不可拖动。
- 图标吸附时必须保留安全边距，避免被浏览器滚动条遮挡。
- 默认边距不得小于 12px，缩边隐藏后的可见把手宽度不得小于 6px。
- 入口 5 分钟无交互且当前处于 `idle` 时，进入 `collapsed-idle` 状态，组件滑向最近边缘但保留把手。
- 鼠标 hover、靠近入口把手，或任意点击入口组件时，图标应完整滑出。
- 入口位置、折叠侧边、主题配置和行为开关都应持久化。

### 中央提示层

- 进入 `selecting` 模式时在屏幕中央显示高可见性的提示层。
- 提示文案至少包含“请选择一段文本”和“Esc 退出”。
- 提示层不能阻挡用户选择文本段；应允许点击穿透到页面候选段落。
- 一旦进入 `typing` 模式或退出模式，提示层立即消失。

### 设置面板

- 设置面板由独立设置按钮展开和关闭，不与开始练习主按钮复用同一点击动作。
- 面板内必须包含：
  - 3 套预设主题：`Classic`、`Soft`、`High Contrast`
  - 5 个高级颜色项：选段框线、未打字、已打对、打错、已跳过
  - `是否始终跟随当前打字段到屏幕中心` 开关
- 切换预设后可覆盖当前颜色值；用户手动改色后应立即反映到当前练习层。
- 设置面板打开时视为“使用中”，不得触发 5 分钟自动缩边。

### 原位打字覆盖层

- 继续使用脚本自身的固定定位覆盖层渲染当前段落，而不是修改目标段落子节点。
- 覆盖层必须尽量继承目标段落的字体、字号、字重、字距、行高和文本对齐。
- 进入练习时目标段落允许临时 `visibility: hidden`，但退出后必须恢复原始 `style` 外观。
- 覆盖层字符状态扩展为：
  - `pending`：未打字
  - `correct`：打对
  - `error`：打错
  - `skipped`：手动跳过
- 错误字符需要有明显错误反馈，默认采用错误文字色加高对比背景块。
- 覆盖层跟随窗口滚动和尺寸变化同步位置。

### 输入、跳过与统计规则

- 仍然按 Unicode 字符序列处理英文、中文、表情和符号。
- 普通输入：
  - 正确时写入 `correct`
  - 错误时写入 `error`
  - 两者都推进当前位置
- `Backspace`：
  - 回退一个当前位置
  - 被回退字符恢复成 `pending`
  - 已发生的错误次数不回滚
- `Tab`：
  - 仅在 `typing` 模式下拦截
  - 当前字符标记为 `skipped`
  - 推进到下一个字符
  - 不增加 `typedCount`
  - 不增加 `errorCount`
- `Shift+Tab`：
  - 仅在 `typing` 模式下拦截
  - 当前段剩余字符全部标记为 `skipped`
  - 显示约 `600ms` 的跳过预览
  - 然后自动切到下一段
  - 不增加 `typedCount`
  - 不增加 `errorCount`
- IME：
  - `compositionstart` 到 `compositionend` 之间不处理中间拼音
  - 提交文本后逐字符比较，并产生 `correct/error` 状态
- 速度统计：
  - `CPM = 实际输入字符数 / 已用分钟数`
  - `WPM = (实际输入字符数 / 5) / 已用分钟数`
  - `错误率 = 错误输入次数 / 实际输入字符数 × 100%`
  - `Tab/Shift+Tab` 跳过字符完全不算入“实际输入字符数”
- 右下角统计需要按时间定时刷新，推荐 `250ms` 一次，而不是只在击键后刷新。

### 跟随滚动

- 新增配置项：`是否始终跟随当前打字段到屏幕中心`。
- 开启时：
  - 切到新段落时自动滚动到屏幕中部附近
  - 若当前段在连续输入过程中明显离开视口，也允许继续进行温和居中
- 关闭时：
  - 不因输入或切段主动滚动页面
  - 覆盖层仅随页面已有滚动同步位置

## 状态模型

脚本 UI 需要至少支持这些显式状态：

- `idle`：入口可见，未开始练习
- `collapsed-idle`：入口缩边隐藏，只留把手
- `selecting`：中央提示显示，框线启用
- `typing`：统计面板显示，覆盖层显示
- `segment-skip-preview`：整段跳过后的短暂反馈状态

`Esc` 在 `selecting`、`typing`、`segment-skip-preview` 中都必须立即退出到 `idle`。

## Tech Stack

- 继续使用 JavaScript ES2020+、单文件 Tampermonkey userscript。
- 继续使用原生 DOM API、Pointer Events、Keyboard/Input/Composition Events、Shadow DOM。
- 新增可测试的配置与计时逻辑仍然保持在原生模块中，不引入 UI 框架。

## Project Structure Impact

在现有结构上，预期这些模块职责会扩展：

- `src/core/session.js`
  增加字符渲染状态、跳过状态和“是否计入统计”的建模
- `src/ui/widget.js`
  增加中央提示层、设置面板层、折叠入口状态、主题应用和更复杂的覆盖层样式
- `src/app.js`
  增加状态切换、空闲计时器、Tab/Shift+Tab 拦截、跟随滚动开关和配置持久化
- `tests/app.test.js`
  补入口折叠、中央提示、Tab/Shift+Tab、实时统计刷新和设置面板交互测试
- `tests/widget.test.js`
  补颜色应用、覆盖层字符状态、提示层和面板显隐测试
- `tests/session.test.js`
  补 `skipped` 状态、统计排除和整段跳过预览测试

## Testing Strategy

### 单元测试

- `TypingSession` 的 `correct/error/skipped/pending` 字符状态流
- `Tab` / `Shift+Tab` 不计入 `typedCount`
- 计时驱动的 WPM/CPM 刷新与零输入边界
- 入口折叠计时器和安全边距计算

### jsdom 集成测试

- 点击主图标进入中央提示层
- 设置按钮展开、切换预设、改色即时生效
- `Tab` 跳过单字符并渲染跳过色
- `Shift+Tab` 跳过整段并在预览后切段
- 关闭跟随后不触发自动滚动
- 右键主图标不进入拖动
- 退出后目标段落 `style` 恢复

### 手工浏览器验收

- 深色、浅色、带滚动条页面的入口边距与缩边隐藏
- 中英文混排、链接段落、表情字符的 `Tab` 跳过和错误高亮
- 设置面板改色后立刻反映到当前练习层
- 5 分钟空闲后的把手可见性与滑出体验

## Boundaries

### Always

- 继续保持目标文本节点结构不被拆分或替换。
- 所有临时样式都必须在退出时完全恢复。
- 所有设置都保存在本地，不联网。
- `Tab`/`Shift+Tab` 拦截只在打字模式内生效。

### Ask First

- 增加更多主题、统计历史、云同步、快捷键自定义。
- 把设置面板做成完整可拖动窗口。
- 为跳过、错误、完成增加声音或动画音效。

### Never

- 在非打字模式下全局屏蔽浏览器原生 `Tab` 行为。
- 让入口图标完全不可见到难以重新发现。
- 修改网页业务输入框或链接本身的文本内容。

## Success Criteria

1. 点击主图标后，中央提示层出现，文案明确提示“选择文本段”和 `Esc` 退出。
2. 设置按钮可独立展开面板，并支持预设主题与 5 个高级颜色配置。
3. 当前段落覆盖层可区分 `pending/correct/error/skipped` 四种字符视觉状态。
4. `Tab` 跳过当前字符，显示跳过色，且不计入速度统计。
5. `Shift+Tab` 跳过当前段剩余字符，显示短暂预览后进入下一段，且不计入速度统计。
6. 右下角速度统计在无击键时也会随时间持续刷新。
7. 跟随滚动开关关闭后，切段不自动滚动页面。
8. 右键图标不能触发拖动；图标吸附保留安全边距。
9. 入口空闲 5 分钟后缩边隐藏，但仍保留可发现把手并可滑出恢复。
10. `npm run verify` 和新增浏览器验收全部通过。
