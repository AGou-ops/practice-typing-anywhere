# Spec: Typing Everywhere 油猴脚本

## 已确认假设

1. 交付物是一个可安装到 Tampermonkey 的单文件 JavaScript 用户脚本。
2. “文本段”按最近的语义文本块识别，首版支持 `p`、`li`、`blockquote`、`pre`、`figcaption`、`h1` 至 `h6`。
3. 用户选择起始文本段后，脚本按 DOM 阅读顺序持续练习后续可见文本段，直到用户按 `Esc`。
4. 脚本不拆分、不替换、不包裹、不着色目标页面的文本节点，也不修改目标文本段的属性或事件。
5. 脚本可以创建并维护自己的图标、选择轮廓、隐藏输入控件和统计弹窗；这些节点必须与页面业务 DOM 隔离。
6. 功能复现以目标扩展公开可观察的核心体验为参考，不复制其源码、品牌资源或未公开实现。

## Objective

为希望利用网页内容练习打字的用户提供一个轻量、非侵入式油猴脚本。脚本在任意普通网站显示一个可拖动并吸附视口边缘的入口图标。用户点击图标后选择一个完整语义文本段，并直接通过键盘开始练习。

练习期间页面原文保持不变。脚本在后台逐字符比较输入，并在右下角弹窗实时显示 WPM、CPM 和错误率。当前段落输入完毕后自动进入下一个语义文本段，持续运行到用户按 `Esc` 退出。

### 用户流程

1. 页面加载后，视口右上角显示入口图标。
2. 用户可拖动图标；释放后图标吸附到最近的视口边缘，并记住位置。
3. 用户点击图标进入选段模式。
4. 鼠标移动到文本上时，脚本用独立覆盖轮廓标示最近的语义文本块，不修改该元素本身。
5. 用户点击文本段，将其设为起始段并进入打字模式。
6. 脚本通过自身的隐藏输入控件接收普通键盘输入和 IME 组合输入。
7. 输入完成当前段落后，自动选择 DOM 阅读顺序中的下一个有效文本段；目标在视口外时滚动到可见区域。
8. 用户按 `Esc` 退出打字模式，统计弹窗消失，入口图标恢复可用。

## Functional Requirements

### 入口图标

- 默认位于视口右上角，不影响网页正常布局。
- 支持鼠标拖动和触摸拖动。
- 释放后吸附到上、下、左、右中距离最近的视口边缘。
- 使用 `localStorage` 持久化归一化位置；窗口尺寸改变后仍保持在视口内。
- 拖动结束不得误触发选段模式。

### 文本段选择

- 候选元素限定为可见、非空的语义文本块。
- 排除脚本自身 UI、表单控件、可编辑区域、隐藏元素、`script`、`style`、`noscript` 及 `aria-hidden="true"` 内容。
- 嵌套元素命中时选择最近的候选语义祖先。
- 悬停提示由脚本自己的固定定位轮廓层实现，不给目标元素添加 class、style 或属性。
- 点击候选段落仅用于确定起点；应阻止这一次点击触发链接或页面业务操作。

### 输入与字符比较

- 目标文本取自元素可见文本，并进行一致的空白归一化。
- 比较单位为 Unicode 字符序列，不能把代理对拆成两个字符。
- 普通输入按字符与当前位置目标字符比较；无论正确或错误，位置都继续前进。
- 错误输入累计到错误次数，后续退格不得撤销已经产生的错误统计。
- `Backspace` 允许将当前位置回退一个字符，但不减少总输入次数或错误次数。
- IME `compositionstart` 到 `compositionend` 期间不判定中间拼音；提交文本后逐字符比较。
- `Esc` 在选段模式或打字模式下均立即退出当前模式。
- 打字模式下阻止练习字符进入网页自身输入框；退出后恢复页面正常键盘行为。

### 连续段落

- 从起始段落开始，按文档中的 DOM 顺序构建后续有效段落序列。
- 动态页面中，切换到下一段前重新验证元素是否仍连接、可见且有文本；失效时跳过。
- 当前段完成后立即进入下一段，不显示完成页、提示按钮或中断确认。
- 到达当时可发现的最后一段后继续等待；DOM 新增有效后续段落时可以继续，否则保持当前统计，直到用户按 `Esc`。
- 下一目标不在视口内时使用平滑滚动使其可见，但不改变页面 DOM。

### 实时统计弹窗

- 仅在打字模式中显示于视口右下角。
- 仅显示 `WPM`、`CPM`、`错误率` 和简短的 `Esc` 退出提示。
- 不显示段落进度、完成状态、重练按钮、退出按钮或结果页。
- 计时从第一次有效输入开始；尚未输入时三个指标显示为 `0`。
- `CPM = 总输入字符数 / 已用分钟数`。
- `WPM = (总输入字符数 / 5) / 已用分钟数`。
- `错误率 = 错误输入次数 / 总输入字符数 × 100%`。
- 指标至少每 250 毫秒刷新一次，并避免每次按键触发高成本 DOM 查询。

## Tech Stack

- JavaScript ES2020+，单文件 Tampermonkey userscript。
- 原生 DOM API、Pointer Events、Keyboard/Input/Composition Events。
- Shadow DOM 隔离脚本 UI 样式，避免与站点 CSS 相互污染。
- 测试使用 Node.js 内置测试运行器和 `jsdom`；不引入 UI 框架或运行时依赖。

## Code Style

- 使用 ESM、`const`/`let`、早返回和小型单职责函数。
- 变量和函数使用 `camelCase`，类使用 `PascalCase`，常量使用 `UPPER_SNAKE_CASE`。
- DOM 查询和业务计算分离；可测试的比较与统计逻辑不得依赖浏览器全局状态。
- 默认使用 ASCII 标识符，用户可见文本使用简体中文。

```js
export function calculateMetrics({ typedCount, errorCount, elapsedMs }) {
  if (typedCount === 0 || elapsedMs <= 0) {
    return { wpm: 0, cpm: 0, errorRate: 0 };
  }

  const minutes = elapsedMs / 60_000;
  return {
    wpm: typedCount / 5 / minutes,
    cpm: typedCount / minutes,
    errorRate: errorCount / typedCount,
  };
}
```

## Testing Strategy

### 单元测试

- 语义文本段识别、排除规则和 DOM 顺序。
- Unicode 字符拆分、空白归一化和逐字符比较。
- 正确、错误、退格、IME 提交后的状态转换。
- WPM、CPM、错误率及零时长边界。
- 图标吸附位置和视口约束计算。

### jsdom 集成测试

- 点击入口、悬停候选、选择段落、连续切段、`Esc` 退出的完整状态流。
- 验证目标文本段的 `innerHTML`、属性和子节点引用在练习前后保持不变。
- 验证脚本 UI 位于独立宿主和 Shadow DOM 中，并在退出时正确清理临时 UI。
- 验证表单控件、可编辑区域和脚本自身 UI 不会被选为练习段落。

### 手工浏览器验收

- 在新闻文章、文档站点、中英文混排页面和含链接的段落中测试。
- 验证拖动、边缘吸附、页面滚动、窗口缩放和动态新增内容。
- 使用中文输入法确认组合期间不误判，提交后统计正确。
- 确认退出后网页点击、键盘和输入框行为完全恢复。

核心计算与状态逻辑要求达到至少 90% 行覆盖率；浏览器兼容性以手工验收补充。

## Boundaries

### Always

- 只读目标页面文本和几何信息，不改写目标文本段 DOM。
- 所有事件监听都可撤销，并在模式退出或脚本销毁时清理。
- 所有站点内容都视为不可信文本，不执行其中的 HTML 或脚本。
- 在构建交付前运行 `npm run verify`。

### Ask First

- 增加设置面板、快捷键配置或统计历史。
- 增加逐字符高亮、声音、排行榜或云同步。
- 引入生产运行时依赖或 UI 框架。
- 改变指标公式、段落识别规则或连续练习行为。

### Never

- 修改、包裹、替换目标文本节点，或向目标段落写入 class、style、属性。
- 捕获密码框、支付输入、普通表单输入或 `contenteditable` 内容用于练习。
- 向远端发送网页文本、用户输入或统计数据。
- 注入到浏览器内部页面、Chrome Web Store、浏览器 PDF 查看器或无法访问的跨域 iframe。
- 复制目标 Chrome 扩展的源码、图标、名称或其他品牌资源。

## Success Criteria

1. Tampermonkey 安装构建产物后，普通网站右上角出现可拖动、可吸附且位置可持久化的入口图标。
2. 点击入口后可选择最近的完整语义文本段，并从该段开始接收普通键盘和 IME 输入。
3. 练习前后目标段落的 HTML、属性、子节点和页面业务事件保持不变。
4. 输入错误时位置继续前进；错误率严格按累计错误输入次数除以总输入次数计算。
5. 右下角弹窗实时显示 WPM、CPM 和错误率，公式与本规格一致。
6. 当前段完成后自动进入后续有效段落，不出现完成页、重练或退出按钮。
7. 用户按 `Esc` 后立即结束练习，移除统计弹窗并恢复网页正常交互。
8. `npm run verify`、覆盖率门禁及手工浏览器验收全部通过。

## Non-Goals

- 不在原文上显示已输入、正确、错误或光标位置。
- 不提供本轮完成、成绩汇总、重练或弹窗退出按钮。
- 不保存历史成绩，不登录，不联网，不同步。
- 不保证支持 Canvas 文本、浏览器 PDF、Shadow DOM 内不可访问文本或跨域 iframe。
- 不追求与参考扩展内部实现或所有附加功能完全一致。
